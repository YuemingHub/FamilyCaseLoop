## 两个缺口（是"补容器"，不是"造逻辑"）

1. **真实案例容器为空**：`familyedu-os/active/private-library/09-client-service-records/active/` **当前不存在/为空**（START.md 说的"真实在服务家庭"未落进去）。S8 的"服务档案"没地方写。这是新仓库/工作台要补的第一个真实资产。
2. **诊断词表过窄**：双维诊断的判定词表过窄（"动太多/动太少"被误判成"未就绪"），S5 引擎要等月明校准信号词才准。

## 自用版 与 同行版 的关系（防重复造）

- **自用版 = `parent-reply-mvp` 这套代码的调试沙盒**：月明自己跑、自己说"可以用了"。
- **同行版 = 同一套代码 + 工作流文档 + 知识库，包成引导前端**：同行（教育者/咨询师/研究者）照着 S0–S8 走通一个个案。
- 两边**共用 S3/S5/S6/S7 引擎**，不重写。

- `yamai-v0.1` 那种"家长直接对话"原型，按 `31` 第1节和 START.md 都属"对外/自媒体"，**不进同行版主线**，要的话单独隔离。
