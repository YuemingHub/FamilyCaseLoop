# 月明家长回复生成器 · MVP

本地运行的家长回复生成微服务，复刻月明老师「双维诊断 + 七层理论 + 安全门」的承接式回复方式。
是 `delivery/` 架构基线中 M2（双维诊断）+ M3（七层透镜）+ M4（MingOS 两级委员会路由）+ M5（parent-reply 话术）的最小纵向切片。

## 能力
- **安全门（最高优先级）**：消亡/自危/暴力/失联信号 → 只做安全承接，不分析原因；含隐性信号与反向否认裁决。
- **高 stakes 灰区交还月明**：离婚/退学/退费/解约/终止服务等 → 不替月明拍板，转交。
- **双维诊断**：承载力状态（崩溃/耗竭/维持/恢复/成长/发呆）× 行为惯性（动太多/动太少/已松动/未就绪），含误判高危区裁决。
- **七层透镜**：依恋/共调节/家庭系统/SDT/权威型/叙事/生态，一次最多 2 层，化进话术不露名词。
- **MingOS 两级委员会路由**：L1 姿态委员会（接住强度）+ L2 内容委员会（动作粒度）。
- **话术生成 + 失败模式硬拦截（FM1-9）**：LLM 走月明 system prompt；无 key 走月明签名动作库确定性降级，发出前强制扫学派名词/指令式/粒度超量。

## 运行
```bash
pip install -r requirements.txt
cp .env.example .env        # 填入 DEEPSEEK_API_KEY（不填也能跑，走离线降级）
python run.py
```
服务地址：http://127.0.0.1:8000 （仅本机，不对外）。

## 调用
```bash
curl -X POST http://127.0.0.1:8000/reply \
  -H 'Content-Type: application/json' \
  -d '{"parent_message":"孩子天天玩手机，我一说他就吼我，我真的快崩溃了"}'
```
返回 `reply`（可直接发家长的回复）+ `diagnosis` + `lens` + `risk_level` + `route` + `fm_violations` + `source`。

- `POST /reply`：生成回复
- `POST /analyze`：返回完整分析（诊断/透镜/路由/违规）
- `GET /health`：健康检查（含 llm_enabled）
- `GET /settings`：返回掩码后的 key / provider / llm_enabled（不泄露明文）
- `POST /settings`：网页端写入 key / provider，持久化到 `.env` 并**立即生效**（无需重启）
- `GET /`：单页可视化工作台（见下）

## 网页可视化工作台

浏览器打开 http://127.0.0.1:8000 即可看到单页工作台，分四块：

1. **① LLM 密钥设置**：输入 API Key（仅存本机 `.env`，掩码显示 `sk-****bcd`，不出本机）；
   选 `deepseek`（云端）或 `ollama`（本机离线）。点「保存并立即生效」即写入并热更新，
   顶部状态徽标实时显示 provider / LLM 是否启用 / key 掩码。
   留空 Key 则自动走「月明签名动作库」确定性降级，无需联网也能出回复。
2. **② 家长原话**：粘贴家长的话，点「生成回复 + 可视化」。
3. **③ 决策可视化**：逐步展示方法论的判定过程——
   风险等级（安全门 / 交还月明 / 常规）→ 双维诊断（承载力×行为惯性+标签）
   → 七层透镜（已化进话术、不露名词）→ MingOS 路由（姿态/内容/动作数/禁步骤清单/邀请式）
   → 失败模式硬拦截（FM 违规列表）。
4. **④ 给家长的回复**：最终可直接发送的回复正文 + 来源（llm / deterministic / 降级）。

> 安全门触发时只做安全承接、不展示诊断；灰区交还时标「已交还月明」并附诊断供月明参考。

## 说明
- 确定性安全门与双维诊断不依赖 LLM，可离线单测（`pytest tests/`）。
- LLM 仅作话术润色；所有安全门/灰区/失败模式拦截为确定性护栏，LLM 无法绕过。
- 本地数据不出本机；敏感案例可切 `LLM_PROVIDER=ollama` 走本地模型。
- 网页设置密钥写入项目根 `.env`（已被 `.gitignore` 排除），密钥不出本机、不进版本库。
- LLM 调用带 20s 超时：key 错误或网络不通时自动降级到签名动作库，不会卡住界面。
