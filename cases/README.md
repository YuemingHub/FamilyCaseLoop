# cases/ · 真实案例容器（S8 落点）

本目录是 **FamilyCaseLoop 同行版** 的「真实案例容器」，对应个案全生命周期链路 `S0–S8` 的最终回流收口节点（S8）。

> 这是缺口①的落点。引擎（`engine/`）负责 S3/S5/S6/S7 的分析与生成；引导前端（`guide/`）在 S8 提供「导出本次分析为个案草稿」与「下载填写模板」两个纯前端动作，把结果落回本目录。**本目录不依赖任何外部大仓路径，同行版完全 decoupled。**

## 目录结构

```
cases/
├── README.md              # 本说明
├── _template/
│   └── CASE_TEMPLATE.md   # 单个个案全链填写模板（S0–S8）
└── active/                # 真实（脱敏）个案存放处 ← 月明把草稿存这里
    └── .gitkeep
```

## ⛔ 脱敏红线（务必遵守）

- **未脱敏的原始个案不得进入本目录、不得进入任何对外/同行链路。**
- 删除所有可识别信息：真实姓名、学校、地名、门牌、具体日期、聊天账号、手机号。
- 用化名 + 年龄段 + 脱敏后的原话片段即可（如「13岁男孩母亲」「孩子说活着没意思（原话已脱敏）」）。
- 未成年个案一律从严；涉及高危（自伤/伤人/被侵害）必须先走 S3 安全门，不得自动放行。

## 如何填写

1. 在 `guide/` 走完 S0–S8：S3 粘贴脱敏后的家长原话 → 运行引擎 → S5/S6/S7 看分析结果 → S8 选主收口与三项同步。
2. 点 S8 面板的 **「导出本次分析为个案草稿」** → 浏览器下载一个 `.md` 文件（含 S0–S8 全链内容）。
3. 把下载的文件**重命名为个案化名**（如 `active/个案_小雨妈_2026Q3.md`）存入 `active/`。
4. 也可先点 **「下载个案填写模板」** 拿空白模板，手动补充 S0/S1/S2/S4/S8 里引擎不覆盖的主观判断。

## 与映射文档的关系

- `docs/case-lifecycle-mapping.md` 里 S8 的「缺口①」即指本容器，现已补建。
- 本容器只存**脱敏后的服务档案**，不存原始录音/文稿；原始材料仍归月明本地工作台管理。

## 提交约定

- `active/` 下的真实个案是否入库由月明决定（含敏感信息的可只留本地、不 push）。
- `README.md` / `_template/` 随仓库版本走，正常提交。
